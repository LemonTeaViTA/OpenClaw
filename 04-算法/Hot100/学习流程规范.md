# 📋 Hot100 学习流程规范

> **目标**：支持不固定时间、不固定进度的灵活学习模式  
> **核心**：每次学习都能准确记住上次进度  
> **ID 系统**：使用 LeetCode 题号作为唯一 ID（如 `lc-1`、`lc-15`），避免题目名称乱码或变更

---

## 🔄 标准学习流程

### 每次学习前的 3 个步骤

```
┌─────────────────────────────────────────┐
│  Step 1: 说"开始学习"                    │
│  → 我读取状态文件，告诉你上次学到哪里     │
│  → 确认你是否完成了上次的题目            │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│  Step 2: 学习新内容                      │
│  → 我提供当日学习材料                   │
│  → 你学习完成后说"学习完毕"              │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│  Step 3: 更新状态                        │
│  → 我标记题目为已完成                   │
│  → 记录复习日期，进入下一天              │
└─────────────────────────────────────────┘
```

## 📅 复习策略

### 艾宾浩斯遗忘曲线

```
学习后复习时间点：
├─ 第 1 天（24 小时后）→ 防止快速遗忘
├─ 第 3 天              → 巩固短期记忆
├─ 第 7 天              → 转入长期记忆
└─ 第 14 天             → 深度巩固
```

### 复习优先级

```
1. 薄弱题目（标记为不掌握）
2. 即将到期的复习（1 天内）
3. 已过期的复习（超期未复习）
4. 正常复习队列
```

## ⚠️ 注意事项

### 1. 每次学习后务必说"学习完毕"
- 这样我才能更新状态
- 否则下次还会认为你没学完

### 2. 如果间隔多天未学习
- 我会自动检查复习队列
- 优先安排过期未复习的题目

### 3. 如果某天学了很多
- 告诉我你学到了哪一天
- 我会批量更新状态

### 4. 如果某天只学了部分
- 告诉我题目没学完
- 状态保持 `in_progress`，下次继续

## 📋 题目 ID 说明

**格式**：`lc-{leetcode_题号}`

**示例**：
- `lc-1` → 两数之和
- `lc-11` → 盛最多水的容器
- `lc-15` → 三数之和
- `lc-42` → 接雨水

**优点**：
- ✅ 唯一性：每道题有且仅有一个 ID
- ✅ 稳定性：不随题目名称变更而变化
- ✅ 简洁性：比题目名称更短，易于记录
- ✅ 无乱码：纯 ASCII 字符，无编码问题

**完整映射表**：见 `题目 ID 映射表.md`
