## 📝 题目一：两数之和（🟢 简单）⭐

### 题目描述

给定一个整数数组 `nums` 和一个整数目标值 `target`，请你在该数组中找出和为目标值的那**两个**整数，并返回它们的数组下标。

**示例：**
```
输入：nums = [2,7,11,15], target = 9
输出：[0,1]
解释：因为 nums[0] + nums[1] == 9，返回 [0, 1]。
```

### 解法一：哈希表（最优解）⭐⭐⭐

**核心思路：**
- 用哈希表存储已遍历的数字及其索引
- 每次检查 `target - num` 是否在哈希表中
- 存在则返回，不存在则存入当前数字

**代码实现：**
```python
def twoSum(nums, target):
    hashmap = {}
    for i, num in enumerate(nums):
        complement = target - num
        if complement in hashmap:
            return [hashmap[complement], i]
        hashmap[num] = i
    return []
```

**复杂度分析：**
- ⏱️ 时间复杂度：O(n)，只需遍历一次
- 💾 空间复杂度：O(n)，哈希表存储 n 个元素

### 解法二：暴力枚举

```python
def twoSum(nums, target):
    n = len(nums)
    for i in range(n):
        for j in range(i + 1, n):
            if nums[i] + nums[j] == target:
                return [i, j]
    return []
```

**复杂度分析：**
- ⏱️ 时间复杂度：O(n²)，会超时
- 💾 空间复杂度：O(1)

### ⚠️ 易错点

1. **不能使用同一个元素两次**
   - 错误：`[3, 3]`, target=6 → 返回 `[0, 0]` ❌
   - 正确：应该返回 `[0, 1]` ✅

2. **先检查再存入**
   - 顺序不能颠倒，否则会匹配到自己

3. **返回索引顺序**
   - 题目没有要求顺序，但通常返回 `[较小索引，较大索引]`

## 📝 题目二：盛最多水的容器（🟡 中等）⭐⭐⭐

### 题目描述

给定一个长度为 n 的整数数组 `height`。有 n 条垂线，第 i 条线的两个端点是 `(i, 0)` 和 `(i, height[i])`。找出其中的两条线，使得它们与 x 轴共同构成的容器可以容纳最多的水。

**示例：**
```
输入：[1,8,6,2,5,4,8,3,7]
输出：49
解释：选择第 2 条线 (高度 8) 和第 9 条线 (高度 7)，
     面积 = min(8, 7) × (9-2) = 7 × 7 = 49
```

### 解法一：双指针（最优解）⭐⭐⭐

**核心思路：**
- 双指针从两端向中间移动
- 面积 = 短板高度 × 宽度
- **每次移动较短的板**（因为面积由短板决定）

**代码实现：**
```python
def maxArea(height):
    left, right = 0, len(height) - 1
    max_area = 0
    while left < right:
        # 计算当前面积
        area = min(height[left], height[right]) * (right - left)
        max_area = max(max_area, area)
        
        # 移动短板
        if height[left] < height[right]:
            left += 1
        else:
            right -= 1
    return max_area
```

**复杂度分析：**
- ⏱️ 时间复杂度：O(n)，双指针遍历一次
- 💾 空间复杂度：O(1)

### 解法二：暴力枚举（会超时）

```python
def maxArea(height):
    max_area = 0
    n = len(height)
    for i in range(n):
        for j in range(i + 1, n):
            area = min(height[i], height[j]) * (j - i)
            max_area = max(max_area, area)
    return max_area
```

**复杂度分析：**
- ⏱️ 时间复杂度：O(n²)，会超时
- 💾 空间复杂度：O(1)

### 💡 核心思想：为什么移动短板？

**反证法理解：**
- 假设当前 `height[left] < height[right]`
- 如果移动长板（right 左移）：
  - 宽度一定减小
  - 高度由短板决定，最大还是 `height[left]`
  - **面积只会更小或不变**
- 如果移动短板（left 右移）：
  - 宽度减小
  - 但高度**可能增加**
  - **面积有机会变大**

**结论：** 只有移动短板，才可能获得更大的面积！

### ⚠️ 易错点

1. **面积计算公式**
   - `area = min(height[left], height[right]) * (right - left)`
   - 别忘了取最小值！

2. **指针移动条件**
   - `height[left] < height[right]` 时移动 left
   - 相等时移动任意一边都可以

3. **循环条件**
   - `while left < right`（不能相等）
