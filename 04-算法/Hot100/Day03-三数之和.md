## 📝 题目：三数之和（🟡 中等）⭐⭐

### 题目描述

给你一个整数数组 `nums`，判断是否存在三元组 `[nums[i], nums[j], nums[k]]` 满足：
- `i != j`、`i != k` 且 `j != k`
- `nums[i] + nums[j] + nums[k] == 0`

返回所有和为 0 且**不重复**的三元组。

**示例：**
```
输入：nums = [-1,0,1,2,-1,-4]
输出：[[-1,-1,2],[-1,0,1]]
```

**注意：**
- 答案中不可以包含重复的三元组
- 返回的三元组内部顺序不限

---

## 💡 解题思路

### 核心思想

将"三数之和"转化为"两数之和"：
1. **固定一个数** `nums[i]`
2. 在剩余数组中找两个数，使它们的和为 `-nums[i]`
3. 使用**双指针**高效查找

### 为什么需要排序？

1. **便于去重**：相同元素相邻，容易跳过
2. **双指针前提**：只有有序才能用双指针
3. **剪枝优化**：可以根据大小关系提前终止

---

## 📖 解法：排序 + 双指针（最优解）⭐⭐⭐

### 代码实现

```python
def threeSum(nums):
    nums.sort()  # 1. 排序
    res = []
    n = len(nums)
    
    for i in range(n):
        # 2. 剪枝：如果当前数 > 0，后面不可能和为 0
        if nums[i] > 0:
            return res
        
        # 3. 去重：跳过重复的起始数
        if i > 0 and nums[i] == nums[i-1]:
            continue
        
        # 4. 双指针查找
        left, right = i + 1, n - 1
        while left < right:
            s = nums[i] + nums[left] + nums[right]
            
            if s == 0:
                res.append([nums[i], nums[left], nums[right]])
                
                # 5. 去重：跳过重复的左指针
                while left < right and nums[left] == nums[left+1]:
                    left += 1
                
                # 6. 去重：跳过重复的右指针
                while left < right and nums[right] == nums[right-1]:
                    right -= 1
                
                # 7. 收缩指针
                left += 1
                right -= 1
            
            elif s < 0:
                left += 1  # 和太小，左指针右移
            else:
                right -= 1  # 和太大，右指针左移
    
    return res
```

### 复杂度分析

- ⏱️ **时间复杂度**：O(n²)
  - 排序：O(n log n)
  - 双层循环：O(n²)
  - 主导项：O(n²)

- 💾 **空间复杂度**：O(1)
  - 忽略排序的栈空间
  - 结果数组不计入空间复杂度

## 🔑 三重去重技巧

这是本题的**核心难点**！

### 第一重：起始数去重

```python
if i > 0 and nums[i] == nums[i-1]:
    continue
```

**为什么？**
- 如果 `nums[i]` 和 `nums[i-1]` 相同
- 那么以 `nums[i]` 起始的三元组一定已经在前一轮出现过
- 必须跳过！

**示例：**
```
[-1, -1, 0, 1, 2]
 ↑   ↑
 i   i+1

第一轮：i=0, nums[i]=-1，找到 [-1, -1, 2] 和 [-1, 0, 1]
第二轮：i=1, nums[i]=-1，如果不去重，会重复找到相同结果！
```

---

### 第二重：左指针去重

```python
while left < right and nums[left] == nums[left+1]:
    left += 1
```

**为什么？**
- 找到一个答案后，如果 `nums[left]` 和下一个相同
- 继续用相同的值会重复

---

### 第三重：右指针去重

```python
while left < right and nums[right] == nums[right-1]:
    right -= 1
```

**为什么？**
- 同理，避免右指针重复

---

## ✂️ 剪枝优化

### 剪枝 1：起始数 > 0

```python
if nums[i] > 0:
    return res
```

**原理：**
- 数组已排序，`nums[i]` 是最小的数
- 如果 `nums[i] > 0`，那么 `nums[left]` 和 `nums[right]` 也都 > 0
- 三个正数相加不可能为 0
- 直接返回！

**效果：** 大幅减少不必要的循环

---

## ⚠️ 易错点

### 1. 去重时机错误

❌ **错误写法：**
```python
# 在找到答案之前就去重
if nums[left] == nums[left+1]:
    left += 1
```

✅ **正确写法：**
```python
# 只有在找到答案后才去重
if s == 0:
    res.append([...])
    while left < right and nums[left] == nums[left+1]:
        left += 1
    # ...
```

**原因：** 提前去重会漏掉正确答案

---

### 2. 去重条件错误

❌ **错误写法：**
```python
if nums[i] == nums[i+1]:  # 和前一个比较，不是后一个！
    continue
```

✅ **正确写法：**
```python
if i > 0 and nums[i] == nums[i-1]:
    continue
```

**原因：** 应该和**前一个**比较，确保不重复使用相同的起始数

---

### 3. 忘记收缩指针

❌ **错误写法：**
```python
if s == 0:
    res.append([nums[i], nums[left], nums[right]])
    # 忘记移动指针，死循环！
```

✅ **正确写法：**
```python
if s == 0:
    res.append([nums[i], nums[left], nums[right]])
    # 去重...
    left += 1   # 必须收缩！
    right -= 1  # 必须收缩！
```
