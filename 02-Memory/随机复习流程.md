# 🎲 随机复习流程

> 版本：1.0  
> 创建时间：2026-03-17  
> 最后更新：2026-03-17

---

## 📋 规则

| 项目 | 说明 |
|------|------|
| **频率** | 每日 1 次 |
| **时间** | 20:00 后 |
| **题库** | Java 基础篇 (56 题) + 集合框架 (29 题) = 85 题 |
| **题数** | 10-15 题（完全随机） |
| **追踪文件** | `.random-review-state.json` |
| **检查机制** | Heartbeat 自动检查 |

---

## 🔄 流程

```
步骤 1：Heartbeat 检查（每 30 分钟）
  → 读取 .random-review-state.json
  → 如果 lastReviewDate < 今天 且 当前时间 > 20:00
  → → 提醒用户完成随机复习

步骤 2：随机抽题
  → 从 85 题中随机抽取 10-15 题
  → 混合随机（不区分章节）
  → 记录题目列表

步骤 3：逐题提问
  → 读题目
  → 用户回答
  → 评分（5 分制）
  → 记录扣分点（如果得分<4 分）

步骤 4：更新记录
  → 更新对应章节的答题记录（添加"随机复习"行）
  → 如果得分<3 分 → 更新 MEMORY.md 薄弱项
  → 更新 .random-review-state.json

步骤 5：完成
  → 统计得分
  → 标记 lastReviewDate = 今天
```

---

## 📝 记录方式

### 1. 章节答题记录（详细）

**示例：** `memory/java-questions/Java 基础篇 - 答题记录.md`

```markdown
## 题 35：String 不可变性

| 日期 | 类型 | 得分 | 扣分点 | 状态 |
|------|------|------|--------|------|
| 03-05 | 正式学习 | 5/5 | - | ✅ |
| 03-15 | 随机复习 | 5/5 | - | ✅ |
| 03-17 | 随机复习 | 4/5 | 漏了"缓存 hash 值" | ✅ |
```

**原则：**
- ✅ 只记录扣分点（1 句话）
- ❌ 不记录完整答题内容
- ❌ 得分≥4 分不记录细节

---

### 2. MEMORY.md（汇总）

```markdown
## 1. 📉 当前薄弱项

| 知识点 | 当前分 | 状态 | 扣分点 | 详细记录 |
|--------|--------|------|--------|----------|
| GC Roots 类型 | 0/5 | 🔴 | 完全忘记 | JVM 篇#题 3 |
```

**原则：**
- ✅ 只记录得分<3 分的题目
- ✅ 索引到章节文件
- ❌ 不重复记录详情

---

### 3. .random-review-state.json（状态追踪）

```json
{
  "lastReviewDate": "2026-03-17",
  "lastQuestions": ["String 不可变性", "HashMap 扩容", ...],
  "lastScore": "4.4/5",
  "nextReviewDate": "2026-03-18",
  "totalReviews": 2,
  "reviewHistory": [
    {
      "date": "2026-03-17",
      "questionCount": 12,
      "score": "4.4/5",
      "sources": ["Java 基础", "集合框架"],
      "weaknesses": ["HashSet 性能 (O(1) 不是 O(log n))"]
    }
  ]
}
```

**原则：**
- ✅ 记录日期、题数、得分
- ✅ 记录薄弱点（用于更新 MEMORY.md）
- ❌ 不记录每题详情

## 📊 评分标准

| 分数 | 标准 | 记录方式 |
|------|------|----------|
| **5/5** | 完整正确 | 只记日期，不记细节 |
| **4/5** | 基本正确，漏 1-2 个点 | 记录扣分点（1 句话） |
| **3/5** | 答对一半，遗漏较多 | 记录扣分点 + 加入薄弱项 |
| **2/5** | 只答出少量要点 | 记录扣分点 + 加入薄弱项 |
| **1/5** | 几乎不会 | 记录"完全忘记" + 加入薄弱项 |
| **0/5** | 完全不会/未回答 | 记录"完全忘记" + 加入薄弱项 |
