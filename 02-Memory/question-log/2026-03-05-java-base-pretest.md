## 第 2 题
**题目：** `short s1 = 1; s1 = s1 + 1;` 对吗？`short s1 = 1; s1 += 1;` 对吗？为什么？

**考点：** 类型转换、复合赋值运算符、隐含强制转换

**用户回答：** [语音回答 22 秒]

**标准答案：** 
- 第一句❌编译错误：`s1+1` 结果是 int 类型，赋值给 short 需要强制转换
- 第二句✅正确：复合赋值运算符隐含强制类型转换，等价于 `s1 = (short)(s1 + 1);`

**评分：** 5/5

## 第 3 题
**题目：** `&` 和 `&&` 有什么区别？（从运算类型、短路特性、使用场景三个角度回答）

**考点：** 逻辑运算符、短路与、NullPointerException 避免

**用户回答：** [语音回答 44 秒]

**标准答案：** 
- 运算类型：`&` 是逻辑与（也可作位运算符），`&&` 是短路与（仅逻辑运算）
- 短路特性：`&` 不短路，两边都计算；`&&` 短路，左边为 false 则右边不计算
- 使用场景：常规逻辑判断推荐用 `&&`，可避免 NullPointerException

**评分：** 5/5

**点评：** 三个维度都回答到位，提到了短路特性和经典应用场景。

## 第 4 题
**题目：** `final`、`finally`、`finalize` 有什么区别？

**考点：** 关键字辨析、异常处理、垃圾回收

**用户回答：** [语音回答 30 秒]

**标准答案：** 
- `final`：修饰符，修饰类→不能继承、修饰方法→不能重写、修饰变量→常量
- `finally`：异常处理，`try-catch-finally` 中总会执行（除非 JVM 退出）
- `finalize`：Object 类方法，对象被 GC 前调用（已废弃，不推荐）

**评分：** 5/5

**点评：** 三个关键字区分清晰，提到了 finalize 已废弃的细节。

## 第 5 题
**题目：** `==` 和 `equals()` 有什么区别？（从比较对象、比较内容、是否可重写三个角度回答）

**考点：** 比较运算符、equals 重写、hashCode 约定

**用户回答：** [语音回答 28 秒]

**标准答案：** 
- 比较对象：`==` 可比较基本类型和引用类型，`equals()` 只能比较引用类型
- 比较内容：`==` 比较值（基本类型）或地址（引用类型），`equals()` 默认比较地址但可重写
- 是否可重写：`==` 不可重写，`equals()` 可重写（String、Integer 等都重写了）
- ⚠️ 规则：重写 equals() 时必须重写 hashCode()

**评分：** 5/5

**点评：** 三个维度回答完整，提到了 String 重写 equals 的经典案例。
