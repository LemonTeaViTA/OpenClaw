## 第 2 题
**题目：** `String str1 = new String("abc")` 和 `String str2 = "abc"` 有什么区别？分别创建了几个对象？

**考点：** 字符串常量池、堆内存、对象数量

**用户回答：** 
> new 一个字符串，首先在字符串常量池里面搜索是否存在相同的字符串，如果存在直接引用，不存在的话就创建然后引用，然后在堆中创建一个然后引用。直接创建字符串的话也是先在常量池中查找，没有创建，有直接引用。这两个不相等，因为一个是常量池一个是堆。

**标准答案：** 
- `new String("abc")`：1 个或 2 个对象（常量池无"abc"时 2 个：常量池 1 个 + 堆 1 个；有时 1 个：仅堆）
- `"abc"`：0 个或 1 个对象（常量池已有时 0 个，无时 1 个）
- `str1 == str2`：false（地址不同）

**评分：** 4/5

**点评：** 常量池机制理解正确，== 比较正确。但未明确说明具体创建对象数量（1 个或 2 个的边界情况）。

## 第 3 题
**题目：** String 是不可变类吗？为什么？字符串拼接是如何实现的？

**考点：** final 修饰、底层数组、拼接原理、不可变好处

**用户回答：** 
> String 是不可变类，因为被 final 修饰符修饰，无法被子类继承，无法被重写方法，同时也没有公开的修改方法，底层数组也是被 final 修饰符修饰，无法修改。字符串拼接可以用加号或者用 StringBuffer 的 append 或者是 String 的 join。
> 
> （补充）不可变的好处就是安全性，因为 String 字符串通常是用来传递信息的，所以为了传递安全，String 不可修改很安全。然后是 hashCode 计算字符串得到 hash 码，字符串不可修改这样就不会改变。还有一个就是字符串常量池缓存，不可修改可以保证可以引用。

**标准答案：** 
- 不可变性：✅ 正确（final 类 + final 数组 + 无修改方法）
- 拼接方式：+（编译为 StringBuilder）、StringBuilder.append()、String.join()
- 不可变好处：安全性、hashCode 缓存、常量池支持、线程安全

**评分：** 5/5

**点评：** 核心要点完整，补充的不可变好处完全正确（安全性、hashCode 缓存、常量池）。

## 第 4 题
**题目：** `Integer a = 127, b = 127` 和 `Integer c = 128, d = 128`，`a == b` 和 `c == d` 的结果分别是什么？为什么？

**考点：** Integer 缓存池、自动装箱

**用户回答：** 
> 缓存池范围 -128-127，a 和 b 是相等的，c 和 d 是不等的，因为 c 和 d 是超出缓存范围的，所以是新建的，因此是不一样。

**标准答案：** 
- `a == b`：true（-128~127 在缓存池内，复用同一对象）
- `c == d`：false（128 超出缓存范围，new 新对象）
- 自动装箱原理：`Integer.valueOf()` 检查缓存池

**评分：** 4/5

**点评：** 缓存池范围正确，结果正确，原因分析正确。补充讲解了自动装箱原理（valueOf 方法）。
