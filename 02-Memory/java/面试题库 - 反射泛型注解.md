## 题目 1：Java 泛型了解么？

**来源：** PDF 题 50 + OPPO 面试  
**难度：** ⭐⭐⭐⭐  
**考察频率：** 🔥🔥🔥 高频

### 题目
Java 泛型了解么？/ 泛型的作用是什么？

### 标准答案要点

**作用：** 提高代码的类型安全

**好处：**
1. 在编译时检查类型一致性
2. 避免不必要的类型转换
3. 避免 ClassCastException

**没有泛型的问题：**
```java
List list = new ArrayList();
list.add("hello");
String str = (String) list.get(0);  // 必须强制转换
```

**有泛型：**
```java
List<String> list = new ArrayList<>();
list.add("hello");
String str = list.get(0);  // 无需强制转换
```

### 面试考察点
- 泛型核心价值理解
- 类型安全意识

## 题目 2：泛型有几种使用方式？

**来源：** PDF 题 50  
**难度：** ⭐⭐⭐  
**考察频率：** 🔥🔥 中频

### 题目
泛型有几种使用方式？

### 标准答案要点

**三种使用方式：**

1. **泛型类**
```java
public class Generic<T> {
    private T key;
    public Generic(T key) { this.key = key; }
    public T getKey() { return key; }
}
Generic<Integer> generic = new Generic<Integer>(123);
```

2. **泛型接口**
```java
public interface Generator<T> {
    public T method();
}
class GeneratorImpl implements Generator<String> {
    public String method() { return "hello"; }
}
```

3. **泛型方法**
```java
public static <E> void printArray(E[] inputArray) {
    for (E element : inputArray) {
        System.out.printf("%s ", element);
    }
}
```

## 题目 3：什么是泛型擦除？

**来源：** PDF 题 50  
**难度：** ⭐⭐⭐⭐⭐  
**考察频率：** 🔥🔥🔥 高频

### 题目
什么是泛型擦除？/ 为什么 Java 泛型是伪泛型？

### 标准答案要点

**定义：** Java 的泛型是**伪泛型**，编译期间所有类型信息都会被擦掉

**本质：**
- 泛型只存在于源码中
- 编译时进行类型检查
- 运行时没有泛型信息

**示例：**
```java
LinkedList<Cat> cats = new LinkedList<Cat>();
LinkedList list = cats;  // 去掉泛型，同一个链表
list.add(new Dog());  // 运行时不检查，完全没问题
```

**为什么需要类型擦除？**
- **向下兼容**：JDK5 之前没有泛型
- 让 JVM 保持向下兼容

### 面试考察点
- 泛型本质理解
- 编译时 vs 运行时

### 题目
说一下你对注解的理解？

### 标准答案要点

**本质：** Java 注解本质上是一个**标记**

**类比：** 生活中一个人的小装扮（戴帽子、戴眼镜）

**特点：**
- 可以标记在类上、方法上、属性上
- 标记自身可以设置值

**作用：** 在编译或运行阶段识别标记，然后做一些事情

**应用场景：**
1. AOP：使用注解作为切点（运行期）
2. Lombok：注解在编译期运行（自动生成代码）
3. Spring：@Autowired 等注解

## 题目 5：注解的生命周期有哪些？

**来源：** PDF 题 51  
**难度：** ⭐⭐⭐⭐  
**考察频率：** 🔥🔥🔥 高频

### 题目
注解的生命周期有哪些？

### 标准答案要点

| 生命周期 | 说明 | 举例 |
|---------|------|------|
| **SOURCE** | 给编译器用的，不会写入 class 文件 | @Override |
| **CLASS** | 会写入 class 文件，类加载阶段丢弃 | - |
| **RUNTIME** | 会写入 class 文件，永久保存，可通过反射获取 | @Autowired |

**举例说明：**

**@Override（SOURCE）：**
- 给编译器检查是否真的重写了父类方法
- class 文件中不会有 Override 标记

**@Autowired（RUNTIME）：**
- 运行时可以通过反射获取注解信息
- 可以拿到标记的值（如 required）

### 面试考察点
- 注解生命周期理解
- 运行时 vs 编译时

## 题目 6：什么是反射？应用场景？原理？

**来源：** PDF 题 52 + 美团/小米/比亚迪面试  
**难度：** ⭐⭐⭐⭐⭐  
**考察频率：** 🔥🔥🔥 高频

### 题目
什么是反射？应用场景？原理？

### 标准答案要点

**定义：** 反射允许 Java 在**运行时**检查和操作类的方法和字段

**作用：** 动态地获取类的字段、方法、构造方法等信息

**反射 vs 普通创建对象：**
```java
// 普通方式（编译时确定）
Person person = new Person();

// 反射方式（运行时确定）
String className = "java.util.Date";
Class<?> cls = Class.forName(className);
Object obj = cls.newInstance();
```

**主要实现类：**
- `java.lang.Class`：核心类
- `java.lang.reflect`包：Method、Field、Constructor

**基本操作：**
```java
// 1. 动态加载类
Class<?> cls = Class.forName("java.util.Date");
Object obj = cls.newInstance();

// 2. 访问方法
Method method = cls.getMethod("getTime");
Object result = method.invoke(obj);

// 3. 访问字段
Field field = cls.getDeclaredField("fastTime");
field.setAccessible(true);  // 私有字段需要
```

**应用场景：**
1. **Spring 框架**：动态加载和管理 Bean
2. **动态代理（AOP）**：创建代理类，运行时动态处理方法调用
3. **测试框架（JUnit）**：发现和执行测试方法（扫描@Test 注解）

**原理：**
- Java 程序编译生成字节码 (.class) 文件
- JVM 类加载时，将类型信息加载进方法区
- 反射就是获取这些信息，进行操作

### 面试考察点
- 反射核心概念
- 实际应用场景（Spring、AOP、测试框架）
- 反射原理理解
