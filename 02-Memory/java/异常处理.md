## 1. 异常处理体系 (题 41)

### Throwable 类层次结构

```
Throwable (所有错误和异常的基类)
├── Error (程序无法处理的严重错误)
│   ├── OutOfMemoryError (内存不足)
│   ├── StackOverflowError (栈溢出)
│   └── 其他 JVM 相关错误
│
└── Exception (程序可以处理的异常)
    ├── 编译时异常 (Checked Exception)
    │   ├── IOException
    │   ├── SQLException
    │   └── 其他必须声明或捕获的异常
    │
    └── 运行时异常 (Runtime Exception)
        ├── NullPointerException
        ├── IndexOutOfBoundsException
        ├── IllegalArgumentException
        └── 其他 RuntimeException 子类
```

### 三大类型对比

| 类型 | 编译时检查 | 处理方式 | 特点 | 举例 |
|------|-----------|---------|------|------|
| **Error** | ❌ 不检查 | 无法处理 | JVM 层面错误，程序无法恢复 | OutOfMemoryError、StackOverflowError |
| **编译时异常** | ✅ 必须处理 | try-catch 或 throws 声明 | 编译器强制要求处理 | IOException、SQLException、ClassNotFoundException |
| **运行时异常** | ❌ 不强制处理 | 建议修复程序逻辑 | 程序逻辑错误导致 | NullPointerException、IndexOutOfBoundsException、ClassCastException |

### 关键理解

- **编译时异常**：如果方法可能抛出，必须：
  - 用 try-catch 捕获，或
  - 用 throws 声明抛出
  - 否则编译不通过

- **运行时异常**：编译器不要求处理，但应该通过修复程序逻辑避免

## 2. 异常处理方式 (题 42)

### 方式一：不处理，直接抛出

```java
public void test() throws Exception {
    throw new Exception("抛出异常");
}
```

- `throws`：声明方法可能抛出的异常（放在方法签名）
- `throw`：实际抛出一个异常对象（放在方法体内）

### 方式二：try-catch 捕获处理

```java
try {
    // 包含可能会出现异常的代码
} catch(Exception e) {
    // 捕获异常并进行处理
} finally {
    // 可选，无论是否异常都会执行的代码
}
```

### ⚠️ 重点：catch 和 finally 同时抛异常

**规则：finally 中的异常会覆盖 catch 中的异常**

```java
public class Example {
    public static void main(String[] args) {
        try {
            throw new Exception("Exception in try");
        } catch (Exception e) {
            throw new RuntimeException("Exception in catch");
        } finally {
            throw new IllegalArgumentException("Exception in finally");
        }
    }
}
```

**结果：** 最终抛出 `IllegalArgumentException`（finally 中的异常），catch 中的异常被丢弃。

### 如何保留 catch 中的异常？

```java
Exception catchException = null;
try {
    throw new Exception("Exception in try");
} catch (Exception e) {
    catchException = e;  // 暂存 catch 中的异常
    throw new RuntimeException("Exception in catch");
} finally {
    try {
        throw new IllegalArgumentException("Exception in finally");
    } catch (IllegalArgumentException e) {
        if (catchException != null) {
            System.out.println("Catch exception: " + catchException.getMessage());
        }
        System.out.println("Finally exception: " + e.getMessage());
    }
}
```

## 3. 经典代码题 (题 43)

### 题目 1：try-catch-finally 执行流程

```java
public class TryDemo {
    public static void main(String[] args) {
        System.out.println(test());
    }
    public static int test() {
        try {
            return 1;
        } catch (Exception e) {
            return 2;
        } finally {
            System.out.print("3");
        }
    }
}
```

**答案：输出 `31`**

**解析：**
1. try 块执行 `return 1`，准备返回 1
2. 没有异常发生，catch 块不执行
3. finally 块必执行，打印 `3`
4. 最后返回 try 中的值 `1`

---

### 题目 2：finally 中有 return

```java
public class TryDemo {
    public static void main(String[] args) {
        System.out.println(test1());
    }
    public static int test1() {
        try {
            return 2;
        } finally {
            return 3;
        }
    }
}
```

**答案：输出 `3`**

**解析：**
- try 返回前先执行 finally
- finally 直接 `return 3`，程序流程从这里返回，走不到 try 里的 return

**⚠️ 警告：** finally 里使用 return 仅存在于面试题中，**实际开发这么写要挨屌的！**

---

### 题目 3：finally 修改变量是否影响返回值

```java
public class TryDemo {
    public static void main(String[] args) {
        System.out.println(test1());
    }
    public static int test1() {
        int i = 0;
        try {
            i = 2;
            return i;
        } finally {
            i = 3;
        }
    }
}
```

**答案：输出 `2`（不是 3！）**

**解析：**
- 在执行 finally 之前，JVM 会先将 `i` 的结果**暂存起来**
- finally 执行完毕后，返回之前暂存的结果
- 所以即使 `i` 被修改为 3，最终返回的还是暂存的 `2`

**关键理解：** return 语句执行时，返回值已经确定并暂存，finally 中修改变量不影响已暂存的返回值。

## 4. 自定义异常 + 异常链（补充）

### 自定义异常

```java
// 自定义编译时异常
public class MyException extends Exception {
    public MyException(String message) {
        super(message);
    }
}

// 自定义运行时异常
public class MyRuntimeException extends RuntimeException {
    public MyRuntimeException(String message) {
        super(message);
    }
}
```

### 异常链（Throwable.getCause()）

```java
try {
    // 底层操作
} catch (SQLException e) {
    throw new RuntimeException("数据库操作失败", e);  // e 作为 cause
}

// 获取原始异常
Throwable cause = exception.getCause();
```
