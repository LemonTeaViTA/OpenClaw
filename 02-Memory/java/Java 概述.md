## 1. 什么是 Java？(题 1)

**定义：** Java 是一门面向对象的编程语言，由 Sun 公司的詹姆斯·高斯林团队于 1995 年推出

**特点：**
- 吸收了 C++ 的优点，但抛弃了容易出错的地方（如指针）
- 自动垃圾回收
- 平台无关（一次编译，处处运行）

**跨平台：** 只需要在对应平台上安装 JDK，就可以在 Windows、macOS、Linux 上运行

## 2. Java 语言的特点 (题 2)

| 特点 | 说明 |
|------|------|
| **面向对象** | 封装、继承、多态 |
| **平台无关性** | "一次编写，到处运行"，可移植性好 |

| **支持 JIT 编译** | 即时编译器，运行时将字节码转换为热点机器码，提高速度 |

## 3. JVM、JDK、JRE 区别 (题 3)

### 包含关系
```
JDK（最完整）
└── JRE
    └── JVM
```

### 详细对比

| 名称 | 全称 | 作用 | 包含内容 |
|------|------|------|---------|
| **JVM** | Java 虚拟机 | 实现跨平台的关键 | 负责将字节码转换为特定平台的机器码并执行 |
| **JRE** | Java 运行时环境 | 运行 Java 程序 | JVM + 必需的库 |
| **JDK** | Java 开发工具包 | 开发 Java 程序 | JRE + 编译器 (javac) + 文档工具 (javadoc) + 字节码工具 (javap) 等 |

**简单说：** JDK 包含 JRE，JRE 包含 JVM

## 4. 跨平台及原理 (题 4)

### 什么是跨平台？

Java 程序一次编译后，可以在多个操作系统上运行

### 原理

**增加了中间件 JVM**：
- JVM 负责将 Java 字节码转换为特定平台的机器码
- 不同操作系统有不同的 JVM 实现
- Java 程序不直接与操作系统交互，而是通过 JVM

## 5. 字节码 (题 5)

### 什么是字节码？

Java 程序经过编译后产生的 **.class 文件**

### Java 程序执行三步

```
源代码 (.java)
   ↓ 编译
字节码 (.class)
   ↓ 解释
机器码
   ↓ 执行
操作系统执行
```

| 步骤 | 说明 |
|------|------|
| **编译** | 将源代码文件 .java 编译成 JVM 可以识别的字节码文件 .class |
| **解释** | JVM 执行字节码文件，将字节码翻译成操作系统能识别的机器码 |
| **执行** | 操作系统执行二进制的机器码 |

## 6. Java 是"编译与解释并存"的语言 (题 6)

### 编译型 vs 解释型

| 类型 | 说明 | 举例 |
|------|------|------|
| **编译型** | 编译器针对特定操作系统，将源代码一次性翻译成机器码 | C、C++ |
| **解释型** | 解释器对源代码逐行解释成机器码并执行 | Python、JavaScript |

### 为什么 Java 是"编译与解释并存"？

**因为 Java 程序需要：**
1. 先将源代码编译成字节码文件（编译）
2. 再解释执行字节码（解释）

**类比：**
- 编译型：找个翻译，等翻译将小说全部翻译成汉语，一次性读完
- 解释型：找个翻译，翻译一段读一段，慢慢把书读完
- Java：先写成标准格式（编译），再逐段翻译执行（解释）
