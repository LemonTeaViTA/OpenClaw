## 📚 知识点总结

### Serial vs ParNew
- Serial：单线程，客户端，Serial Old 搭配
- ParNew：多线程，服务器，CMS 搭配
- 都用于新生代，标记 - 复制算法

### Parallel Scavenge
- 吞吐量优先 = 用户代码时间 / 总时间
- 后台计算、科学计算首选
- 搭配 Parallel Old

### CMS 收集器
- 4 阶段：初并发重并发（2 个 STW，2 个并发）
- 老年代，标记 - 清除算法
- 缺点：碎片、浮动垃圾、CPU 敏感

### G1 收集器
- Region 分区，逻辑分代
- 标记 - 整理，无碎片
- 可预测停顿，大堆（>4GB）首选
- 4 阶段：初并最筛（3 个 STW，1 个并发）

### 收集器搭配
- ✅ Serial + Serial Old
- ✅ ParNew + CMS
- ✅ Parallel Scavenge + Parallel Old
- ✅ G1（独立）
- ❌ Parallel Scavenge + CMS（目标冲突）

### 收集器选择决策树
1. 堆 > 4GB？→ G1
2. 需要可预测停顿？→ G1
3. 多核（≥8）？→ G1
4. 低延迟 + 小堆？→ CMS
5. 吞吐量优先？→ Parallel
