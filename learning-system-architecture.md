## 📋 架构概述

本系统分为三个核心模块，目标是在面试准备阶段高效复习、针对性突破。

```
┌─────────────────────────────────────────────────────────┐
│                    学习系统 v2.0                         │
├─────────────────────────────────────────────────────────┤
│  ┌─────────────┐   ┌─────────────┐   ┌─────────────┐   │
│  │   薄弱项    │   │    复习     │   │    面试     │   │
│  │  (手动标记)  │   │  (项目 60%  │   │  (简历完成后)│   │
│  │             │   │   八股 40%) │   │  模拟问答   │   │
│  └─────────────┘   └─────────────┘   └─────────────┘   │
│                                                          │
│  ┌─────────────────────────────────────────────────┐   │
│  │              算法 (Hot100)                      │   │
│  │           (独立模块，每日 2-3 题)                │   │
│  └─────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────┘
```

---

## 🎯 模块零：算法（Hot100）

### 功能说明
独立算法学习模块，专门刷 LeetCode Hot100 题目。

### 学习目标
- **题量**：100 题（简单 30 + 中等 50 + 困难 20）
- **频率**：每日 2-3 题
- **分类**：数组、链表、栈队列、哈希表、二叉树、递归回溯、动态规划、贪心、二分查找、滑动窗口

### 学习流程
1. 按分类逐个突破
2. 记录解题思路和复杂度
3. 标记薄弱题目（二刷、三刷）
4. 艾宾浩斯复习（1 天/3 天/7 天/15 天）

### 数据存储
- **位置**：`04-算法/Hot100/`
- **记录**：每道题的解题思路、代码、复杂度分析

## 🎯 模块一：薄弱项

### 功能说明
专门记录和追踪用户手动标记的薄弱知识点。

### 标记规则
| 触发条件 | 说明 |
|----------|------|
| 用户说"不会" | 标记为薄弱项 |
| 用户说"忘记了" | 标记为薄弱项 |
| 用户明确表示不懂 | 标记为薄弱项 |

### 注意事项
- ❌ **不再使用正确率自动判定**（之前出现过问题）
- ✅ **完全依赖用户手动反馈**

### 数据结构示例
```markdown
## 薄弱项清单

- [ ] 知识点 A - 标记时间：2026-03-27
- [ ] 知识点 B - 标记时间：2026-03-27
- [ ] 知识点 C - 标记时间：2026-03-27
```

## 📚 模块二：复习

### 复习类型与优先级

| 类型 | 优先级 | 时间分配 | 说明 |
|------|--------|----------|------|
| **项目复习** | 高 | 60% | 复习用户做过的项目内容 |
| **八股复习** | 低 | 40% | 基础知识、常见面试题 |

### 项目复习
- 内容来源：用户实际参与的项目
- 复习重点：
  - 项目架构设计
  - 技术选型理由
  - 遇到的挑战及解决方案
  - 个人贡献部分

### 八股复习
- 内容来源：常见面试题库
- 复习重点：
  - 计算机基础（网络、操作系统、数据库）
  - 编程语言特性
  - 框架原理
  - 系统设计基础

### 复习策略
1. 优先安排项目复习（60% 时间）
2. 八股复习作为补充（40% 时间）
3. 薄弱项穿插在两者中优先复习

## 🎤 模块三：面试

### 启动条件
- ⏳ **等待简历完成后再启用**
- 原因：面试官会围绕简历内容进行提问

### 功能形式
- **模拟问答**：模拟真实面试场景
- 内容基于简历中的：
  - 项目经历
  - 技术栈

              ↓

    └─────────────────┘
