## 📚 知识点总结

### jstat（JVM Statistics）
- **用途**：监控 GC 统计信息
- **参数**：
  - `-gc`：容量和使用量（KB）
  - `-gcutil`：使用率百分比（%）
  - `-class`：类加载情况
- **命令**：`jstat -gc <pid> 1000 10`

### jmap（Memory Map）
- **用途**：生成堆转储文件
- **命令**：`jmap -dump:live,file=heap.hprof <pid>`
- **场景**：内存泄漏、OOM 排查
- **注意**：生成 dump 会 STW

### jstack（Stack Trace）
- **用途**：生成线程快照
- **命令**：`jstack -l <pid>`
- **场景**：死锁检测、线程阻塞、CPU 100%
- **死锁检测**：grep "deadlock"

### jcmd（Diagnostic Command）
- **用途**：多功能诊断工具（JDK 7+）
- **常用命令**：
  - `GC.heap_dump /path/to/file.hprof`
  - `Thread.print`
  - `GC.run`
  - `VM.flags`
- **优势**：集成 jstat/jmap/jstack 功能

### VisualVM vs JConsole
| 特性 | VisualVM | JConsole |
|------|----------|----------|
| 来源 | 独立下载 | JDK 自带 |
| 功能 | 全面（堆分析、线程分析、插件） | 基础（仅监控） |
| 场景 | 开发环境分析 | 快速监控 |

### 工具选择决策树
```
CPU 100% → jstack（查线程栈）
内存泄漏 → jmap（生成 dump）+ VisualVM（分析）
监控 GC → jstat（实时统计）
线程死锁 → jstack（检测 deadlock）
```

### GC 日志分析
```
[GC (Allocation Failure) ...] → 新生代 GC
[Full GC (Ergonomics) ...] → 老年代 GC
```
- **Allocation Failure**：新生代无法分配
- **Ergonomics**：JVM 自适应决策
- **PSYoungGen**：Parallel Scavenge
- **ParOldGen**：Parallel Old
