## 🎯 两种模式

| 模式 | 触发词 | 流程 | 目标 |
|------|--------|------|------|
| **🎓 学习模式** | "学习"、"新内容"、"Day X" | 先讲解 → 再提问 | 掌握新知识 |
| **📚 复习模式** | "复习"、"考我"、"测试" | 先提问 → 再追问 → 补充讲解 | 巩固旧知识 |

---

## 1️⃣ 🎓 学习模式（新知识）

### 触发场景
- JVM 篇 Day 1-10（全新知识）
- 并发编程篇（全新知识）
- 未学过的章节

### 流程

```
步骤 1：明确学习内容
  → 确认今天要学的章节/题数
  → 例如："JVM Day 3，垃圾回收器，10 题"

步骤 2：逐题讲解（不提问）
  → 读题目
  → 讲解答案（要点 + 记忆口诀 + 类比）
  → 确认理解（"理解了吗？"）
  → 用户说"理解"→ 继续下一题

步骤 3：全部讲解完成后 → 提问检验
  → 随机抽取 3-5 题提问
  → 用户回答
  → 评分（5 分制）
  → 记录薄弱点

步骤 4：记录学习结果
  → 更新章节答题记录
  → 更新 MEMORY.md 薄弱项（如有）
  → 更新 .context-snapshot.json
  → 设置复习点（艾宾浩斯：1 天/3 天/7 天/15 天）

步骤 5：询问是否继续
  → "今天学习完成，是否继续下一章节？"
  → 如果用户说"结束"→ 生成学习总结
```

### 示例

```
AI：现在开始 JVM Day 3，垃圾回收器，共 10 题。先讲解，讲完再提问。

AI：题 1：Serial 和 ParNew 收集器有什么区别？
    【讲解】
    - Serial：单线程，STW，简单高效
    - ParNew：Serial 的多线程版本，其他一样
    - 记忆：Serial=单人干活，ParNew=多人干活
    理解了吗？

用户：理解

AI：题 2：Parallel Scavenge 收集器的特点？
    【讲解】
    - 多线程，吞吐量优先
    - 适合后台计算任务
    - 记忆：Parallel=吞吐量
    理解了吗？

用户：理解

...（10 题全部讲解完成）

AI：10 题讲解完成，现在开始提问检验。

AI：题 1：Serial 和 ParNew 的区别？
用户：Serial 是单线程，ParNew 是多线程...

AI：评分：4/5 ✅ 漏了"ParNew 主要用于新生代"
     记录到答题记录。

...（提问 3-5 题）

AI：今天学习完成，得分 4.2/5。
     薄弱点：CMS 阶段（0/5）→ 加入 MEMORY.md
     复习点：03-16（第 1 次）、03-18（第 2 次）
     是否继续下一章节？
```

## 2️⃣ 📚 复习模式（旧知识）

### 触发场景
- 艾宾浩斯复习点到期
- 随机复习（每日 20:00 后）
- 用户主动要求"考我"、"复习"

### 流程

```
步骤 1：确认复习内容
  → 读取 .context-snapshot.json 或 .random-review-state.json
  → 确认今天要复习的题目
  → 例如："JVM Day 1 第 2 次复习，9 题"

步骤 2：逐题提问（不讲解）
  → 读题目
  → 等待用户回答
  → 用户回答 → 评分（5 分制）
  → 记录扣分点/遗漏点

步骤 3：追问缺漏（如果得分<4 分）
  → 指出遗漏点
  → 追问："XXX 是什么？"
  → 用户再回答
  → 再评分

步骤 4：补充讲解（如果用户说"忘记了"）
  → 讲解正确答案
  → 给记忆口诀
  → 代码题：给代码示例（用户手机无法敲代码）

步骤 5：全部复习完成 → 总结
  → 统计得分
  → 更新薄弱项（得分<3 的题目）
  → 设置下次复习点
  → 更新记录

步骤 6：询问是否继续
  → "复习完成，得分 4.3/5。是否继续学习新内容？"
```

### 示例

```
AI：现在开始 JVM Day 1 第 2 次复习（艾宾浩斯·3 天后），共 9 题。
     先提问，答完再讲解。

AI：题 1：JVM 的作用是什么？
用户：将字节码转换成机器码...

AI：评分：4/5 ✅ 漏了"跨平台"
     追问：JVM 如何实现跨平台？
用户：每个平台有自己的 JVM 实现...

AI：评分：5/5 ✅ 完整

AI：题 3：GC Roots 有哪些类型？
用户：栈... 静态变量... 忘了

AI：评分：2/5 ❌ 漏了"常量、Native"
     【补充讲解】
     - GC Roots 类型：栈 + 静 + 常 + 本
     - 栈：栈帧中的本地变量表
     - 静：静态变量
     - 常：常量
     - 本：Native 方法
     - 记忆口诀：栈 + 静 + 常 + 本
     记录为薄弱项。

...（9 题完成）

AI：复习完成，总分 4.3/5。
     薄弱项：GC Roots 类型（2/5）→ 加入 MEMORY.md
     下次复习：03-18（第 2 次）
     是否继续学习新内容？
```

## 3️⃣ 🎲 随机复习流程（每日）

### 触发条件
- 时间：每日 20:00 后
- 频率：每日 1 次
- 题库：Java 基础篇 (56 题) + 集合框架 (29 题) = 85 题
- 题数：10-15 题（完全随机）

### 流程

```
步骤 1：Heartbeat 检查
  → 读取 .random-review-state.json
  → 如果 lastReviewDate < 今天 且 时间>20:00
  → → 提醒用户完成随机复习

步骤 2：随机抽题
  → 从 85 题中随机抽取 10-15 题
  → 混合随机（不区分章节）

步骤 3：逐题提问
  → 读题目
  → 用户回答
  → 评分（5 分制）
  → 如果得分<4 分 → 记录扣分点（1 句话）
  → 如果得分≥4 分 → 只记日期和得分

步骤 4：更新记录
  → 更新对应章节的答题记录（添加"随机复习"行）
  → 如果得分<3 分 → 更新 MEMORY.md 薄弱项
  → 更新 .random-review-state.json

步骤 5：完成
  → 统计得分
  → 标记 lastReviewDate = 今天
```

### 示例

```
AI（20:00 后）：随机复习时间到！今天从 85 题中抽取 12 题。

AI：题 1：String 不可变性的好处？
用户：线程安全、缓存 hash 值...

AI：评分：5/5 ✅ 完整
     记录：Java 基础篇 - 答题记录.md#题 35

AI：题 2：HashMap 扩容机制？
用户：容量达到阈值时扩容，扩容为原来的 2 倍...

AI：评分：4/5 ✅ 漏了"扩容时 rehash"
     记录：集合框架篇 - 答题记录.md#题 12

...（12 题完成）

AI：随机复习完成，总分 4.4/5。
     薄弱点：无（所有题目≥3 分）
     记录已更新，明天继续！
```

## 4️⃣ 📝 代码题特殊处理

### 场景
- 用户用手机，无法敲代码
- 题目要求写代码（如："写一个单例模式"）

### 处理方式

```
步骤 1：用户口述思路
  → "说说你的思路"
  → 用户描述核心逻辑

步骤 2：AI 评分思路
  → 思路正确 → 4/5
  → 思路有误 → 指出错误

步骤 3：AI 给代码示例
  → "我给你代码示例，帮助理解"
  → 给完整代码 + 注释

步骤 4：用户理解确认
  → "理解了吗？"
  → 用户说"理解"→ 继续
```

### 示例

```
AI：题：写一个线程安全的单例模式（双重检查锁）

用户：我知道要用 volatile 和 synchronized，但是手机没法敲代码...

AI：没事，说说你的思路。

用户：先检查实例是否为 null，如果是就加锁，然后再检查一次...

AI：评分：4/5 ✅ 思路正确
     漏了"volatile 防止指令重排序"

AI：【代码示例】
```java
public class Singleton {
    // volatile 防止指令重排序
    private static volatile Singleton instance;
    
    private Singleton() {}
    
    public static Singleton getInstance() {
        if (instance == null) {  // 第一次检查
            synchronized (Singleton.class) {
                if (instance == null) {  // 第二次检查
                    instance = new Singleton();
                }
            }
        }
        return instance;
    }
}
```
     理解了吗？

用户：理解了

AI：记录：得分 4/5，扣分点"漏了 volatile 作用"
```

## 5️⃣ 📊 评分标准（5 分制）

| 分数 | 标准 | 操作 |
|------|------|------|
| **5/5** | 完整正确，无遗漏 | 只记日期，不记细节 |
| **4/5** | 基本正确，漏 1-2 个点 | 记录扣分点（1 句话） |
| **3/5** | 答对一半，遗漏较多 | 记录扣分点 + 补充讲解 |
| **2/5** | 只答出少量要点 | 记录扣分点 + 完整讲解 |
| **1/5** | 几乎不会 | 记录"完全忘记" + 完整讲解 |
| **0/5** | 完全不会/未回答 | 记录"完全忘记" + 完整讲解 |
