## 🎯 随机复习规则

### Java 基础篇（56 题）

- **每天抽取：** 5-8 题
- **覆盖章节：** Java 概述、基础语法、面向对象、String、异常、IO、反射
- **规则：** **完全随机**，所有题目平等概率
- **记录：** 如果得分<4，自动加入薄弱项池

### 集合框架篇（29 题）

- **每天抽取：** 5-7 题
- **覆盖章节：** ArrayList、HashMap、ConcurrentHashMap、Set、队列
- **规则：** **完全随机**，所有题目平等概率
- **记录：** 如果得分<4，自动加入薄弱项池

## 📊 评分标准（严格版）

| 评分 | 标准 |
|------|------|
| **5/5** | 完整准确，有扩展 |
| **4/5** | 核心点答出，小遗漏 |
| **3/5** | 答出一半，需要追问 |
| **2/5** | 只答出少量，薄弱 |
| **1/5** | 几乎不会 |
| **0/5** | 完全不知道 |

## 🎯 薄弱项复习规则

### 薄弱项池

- **来源：** 所有历史得分<4 的题目
- **管理：** 单独维护，不混入随机复习
- **目标：** 专项突破，直到真正掌握

### 规则

- **复习频率：** 每周 2-3 次（周一、三、五）
- **每次题数：** 3-5 题
- **评分标准：** 严格评分（5/4/3/2/1/0）
- **追问规则：** 评分<4 分，追问到底
- **移出条件：** 连续 2 次得 5 分 → 移出薄弱项池

## 🎯 面试追问模式

### 流程

| 步骤 | 操作 | 目的 |
|------|------|------|
| 1. 提问 | 每次只问 1 个问题 | 模拟真实面试 |
| 2. 回答 | 口述回答 | 检验掌握度 |
| 3. 评分 | **严格评分**（不放水） | 知道差距 |
| 4. 追问 | 针对遗漏点追问 | 引导思考 |
| 5. 补充 | 最后给完整答案 | 加深印象 |
| 6. 记录 | 写入 MEMORY.md | 后续复习 |

### 追问规则

**如果评分 <4 分：**
1. 指出遗漏点
2. 追问相关知识点
3. 直到你答出完整答案
4. 最后给标准答案

**如果评分 =5 分：**
- 简单点评，进入下一题
