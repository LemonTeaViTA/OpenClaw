## 📝 学习方法

### 1. 概念学习

**步骤**：
1. 阅读详解文档（30 分钟）
2. 画出核心概念图（10 分钟）
3. 口述核心要点（5 分钟）

**示例**（RAG）：
```
RAG 核心图：

用户问题 → Embedding → 向量检索 → Top-K 文档 → LLM → 答案
              ↑           ↑
         向量数据库   相似度计算
```

---

### 2. 刷题巩固

**步骤**：
1. 不看答案，尝试回答（5 分钟/题）
2. 对照参考答案，找出差距
3. 记录掌握程度（⭐⭐⭐⭐⭐）
4. 标记薄弱点

**技巧**：
- 用手机录音自己的回答
- 回放找问题
- 重点记忆关键词

## 🎯 面试准备

### 高频考点

1. **RAG vs Fine-tuning**
   - 什么时候用 RAG？
   - 什么时候用 Fine-tuning？
   - 能否结合使用？

2. **Agent 规划方法**
   - CoT（Chain of Thought）
   - ToT（Tree of Thoughts）
   - ReAct（Reasoning + Acting）

3. **Embedding 选择**
   - 中文 vs 英文
   - 维度选择
   - 本地部署 vs API

### 项目关联

**派聪明项目中的 AI 技术**：

```
派聪明 = RAG + Agent + Embedding

1. 用户上传文档 → Embedding → 向量数据库
2. 用户提问 → Embedding → 检索相似文档
3. RAG → LLM 基于文档生成答案
4. Agent → 可以调用工具（搜索、计算等）
```

**面试时这样说**：
> "在派聪明项目中，我负责实现了 RAG 架构。我们使用 BGE 模型将文档转换为 Embedding，存储在 Milvus 向量数据库中。用户提问时，先检索 Top-5 相似文档，然后用 Claude 生成答案。这样既减少了幻觉，又能利用最新文档。"
