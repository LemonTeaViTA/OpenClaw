## 📖 核心概念

### 什么是 Agent？

**Agent = AI 智能体**

Agent 是能够**感知环境、做出决策、执行动作**的 AI 系统。

**核心思想**：让 AI 从"被动回答问题"变成"主动完成任务"。

---

## 🔍 Agent vs 普通 LLM

| 特性 | 普通 LLM | Agent |
|------|---------|-------|
| **交互方式** | 问答式 | 任务式 |
| **记忆能力** | 无（每次对话独立） | 有（长期记忆） |
| **工具使用** | 无 | 可以调用工具 |
| **规划能力** | 无 | 可以分解任务 |
| **自主性** | 被动回答 | 主动执行 |

**类比**：
- 普通 LLM = 百科全书（被动查询）
- Agent = 私人助理（主动完成任务）

## 📊 Agent 工作流程

### 步骤 1：感知（Perception）

```
用户输入："帮我分析上周的销售数据，找出下降原因"
    ↓
Agent 理解：
- 任务类型：数据分析
- 时间范围：上周
- 目标：找出下降原因
```

---

### 步骤 2：规划（Planning）

```
任务分解：
1. 获取上周销售数据（调用数据库 API）
2. 计算环比变化（数学计算）
3. 分析下降原因（数据对比）
4. 生成报告（文本生成）
```

**规划方法**：
- **Chain of Thought（CoT）**：逐步推理
- **Tree of Thoughts（ToT）**：多路径探索
- **ReAct**：推理 + 行动交替

---

### 步骤 3：记忆（Memory）

```
短期记忆：
- 当前对话历史
- 用户偏好

长期记忆：
- 向量数据库存储历史任务
- 用户画像
- 领域知识库
```

---

### 步骤 4：工具使用（Tool Use）

```
可用工具：
- 数据库连接器（查询销售数据）
- Python 解释器（数据分析）
- 搜索引擎（查找行业趋势）
- 文件读写（保存报告）
```

---

### 步骤 5：行动（Action）

```
执行动作：
1. 调用数据库 API → 获取销售数据
2. 执行 Python 代码 → 计算环比
3. 对比分析 → 找出下降原因
4. 生成报告 → 返回给用户
```

## 🎯 Agent 的应用场景

### 场景 1：数据分析助手

**案例**：销售数据分析

- **感知**：理解用户需求（分析销售数据）
- **规划**：获取数据 → 计算 → 分析 → 报告
- **工具**：数据库 API、Python 解释器
- **行动**：生成分析报告

**优势**：
- ✅ 自动化数据处理
- ✅ 减少人工分析时间
- ✅ 提供深度洞察

---

### 场景 2：客服 Agent

**案例**：电商客服

- **感知**：理解用户问题（订单、物流、售后）
- **规划**：查询订单 → 检查物流 → 处理售后
- **工具**：订单系统、物流 API、退款系统
- **行动**：解决问题或转人工

**优势**：
- ✅ 7×24 小时在线
- ✅ 处理常见问题（80%）
- ✅ 复杂问题转人工

---

### 场景 3：代码助手

**案例**：编程助手

- **感知**：理解编程需求（写功能、改 bug）
- **规划**：分析代码 → 编写/修改 → 测试
- **工具**：代码编辑器、编译器、测试框架
- **行动**：提交代码或修复建议

**优势**：
- ✅ 提高开发效率
- ✅ 减少低级错误
- ✅ 代码审查辅助

---

## ⚠️ Agent 的挑战

### 挑战 1：规划错误

**问题**：
- 任务分解不合理
- 执行顺序错误
- 死循环

**解决方案**：
- 使用 CoT/ToT 等规划方法
- 添加执行监控
- 设置最大步数限制

---

### 挑战 2：工具调用失败

**问题**：
- API 返回错误
- 参数格式不对
- 权限不足

**解决方案**：
- 错误重试机制
- 参数验证
- 降级方案（转人工）

---

### 挑战 3：记忆管理

**问题**：
- 上下文超出限制
- 重要信息遗忘
- 记忆冲突

**解决方案**：
- 向量数据库存储长期记忆
- 记忆压缩（摘要关键信息）
- 记忆优先级管理

## 📝 面试原题

### 题 1：什么是 Agent？

**参考答案**：
> Agent = AI 智能体，是能够感知环境、做出决策、执行动作的 AI 系统。与普通 LLM 相比，Agent 有记忆能力、可以使用工具、可以规划任务，从"被动回答问题"变成"主动完成任务"。核心组件包括感知、规划、记忆、工具使用、行动。

**评分标准**：
- ✅ 核心定义（感知 + 决策 + 行动）
- ✅ 与普通 LLM 的区别（记忆、工具、规划）
- ✅ 核心组件（5 个）

---

### 题 2：Agent 的工作流程是什么？

**参考答案**：
> 1. **感知**：理解用户意图、环境状态
> 2. **规划**：分解任务、制定策略（CoT/ToT/ReAct）
> 3. **记忆**：短期记忆（对话历史）+ 长期记忆（向量数据库）
> 4. **工具使用**：调用 API、执行代码、搜索网络
> 5. **行动**：执行动作、返回结果

**评分标准**：
- ✅ 五步流程（感知、规划、记忆、工具、行动）
- ✅ 每步有简短说明

---

### 题 3：Agent 的应用场景有哪些？

**参考答案**：
> 1. **数据分析助手**：自动获取数据 → 分析 → 生成报告
> 2. **客服 Agent**：理解问题 → 查询系统 → 解决问题
> 3. **代码助手**：理解需求 → 编写/修改代码 → 测试

**评分标准**：
- ✅ 至少说出 2 个场景
- ✅ 每个场景有工作流程说明

---

### 题 4：Agent 的挑战有哪些？

**参考答案**：
> 1. **规划错误**：任务分解不合理、执行顺序错误、死循环
> 2. **工具调用失败**：API 错误、参数不对、权限不足
> 3. **记忆管理**：上下文超限、遗忘、记忆冲突
>
> **解决方案**：CoT/ToT 规划、错误重试、向量数据库存储记忆

**评分标准**：
- ✅ 至少说出 2 个挑战
- ✅ 有对应的解决方案思路
