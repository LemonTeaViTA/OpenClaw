## 📖 核心概念

### 什么是 Token？

**Token 是 AI 模型处理文本的基本单位**。

AI 模型（如 GPT、Claude 等）不直接处理原始文本，而是将文本拆分成一个个 Token，然后对 Token 进行编码和计算。

---

## 🔍 Token 的拆分规则

### 英文 Token 拆分

| 原文 | Token 拆分 | Token 数 |
|------|----------|---------|
| `Hello` | `Hello` | 1 |
| `Hello world` | `Hello` + ` world` | 2 |
| `unbelievable` | `un` + `believ` + `able` | 3 |
| `I'm` | `I` + `'m` | 2 |

**规则**：
- 常见单词 = 1 个 Token
- 复杂单词 = 多个子词（Subword）
- 标点符号 = 独立 Token
- 空格通常附着在前一个 Token 上

### 中文 Token 拆分

| 原文 | Token 拆分 | Token 数 |
|------|----------|---------|
| `你好` | `你` + `好` | 2 |
| `人工智能` | `人工` + `智能` | 2 |
| `OpenClaw` | `Open` + `Claw` | 2 |

**规则**：
- 汉字通常 1-2 个字 = 1 个 Token
- 英文单词按子词拆分
- 标点符号独立计算

---

## 📊 Token 与字符的关系

### 估算规则

| 语言 | 估算比例 | 示例 |
|------|---------|------|
| **英文** | 1 Token ≈ 4 个字符 | 1000 Token ≈ 4000 字符 ≈ 750 单词 |
| **中文** | 1 Token ≈ 1.5 个汉字 | 1000 Token ≈ 1500 汉字 |

**经验公式**：
- 英文：`Token 数 ≈ 单词数 × 1.3`
- 中文：`Token 数 ≈ 汉字数 ÷ 1.5`

## 💰 Token 计费模式

### 主流 AI API 计费

| 服务商 | 输入价格 | 输出价格 | 说明 |
|--------|---------|---------|------|
| **GPT-4** | $0.03/1K Token | $0.06/1K Token | 输出更贵 |
| **GPT-3.5** | $0.001/1K Token | $0.002/1K Token | 便宜 10 倍 |
| **Claude 3** | $0.003/1K Token | $0.015/1K Token | 输出贵 5 倍 |
| **文心一言** | ¥0.012/1K Token | ¥0.012/1K Token | 输入输出同价 |

**计费公式**：
```
总费用 = (输入 Token 数 × 输入单价) + (输出 Token 数 × 输出单价)
```

**示例**：
- 输入：1000 Token（约 1500 汉字）
- 输出：500 Token（约 750 汉字）
- GPT-4 费用：`(1000 × 0.03/1000) + (500 × 0.06/1000) = $0.06`

## ⚠️ Token 限制

### 上下文窗口（Context Window）

| 模型 | 最大 Token 数 | 相当于 |
|------|------------|--------|
| **GPT-4** | 8K / 128K | 6 万字 / 100 万字 |
| **GPT-3.5** | 16K | 12 万字 |
| **Claude 3** | 200K | 150 万字 |
| **文心一言** | 128K | 100 万字 |

**影响**：
- 超过限制会丢失最早的内容
- 长文档需要分段处理
- 对话历史需要定期清理

## 🎯 优化 Token 使用

### 节省 Token 的技巧

1. **精简 Prompt**
   - ❌ "请你帮我分析一下这段代码有什么问题，我觉得可能是..."
   - ✅ "分析代码问题："

2. **使用缩写**
   - ❌ "Retrieval-Augmented Generation"
   - ✅ "RAG"

3. **避免冗余**
   - ❌ "我觉得可能是这样的，你觉得呢？"
   - ✅ "是否正确？"

4. **代码压缩**
   - 移除注释
   - 简化变量名（在可理解前提下）

## 📝 面试原题

### 题 1：什么是 Token？

**参考答案**：
> Token 是 AI 模型处理文本的基本单位。AI 不直接处理原始文本，而是将文本拆分成 Token 进行编码和计算。英文中 1 个 Token 约等于 4 个字符或 0.75 个单词，中文中 1 个 Token 约等于 1.5 个汉字。

**评分标准**：
- ✅ 核心定义（基本单位）
- ✅ 拆分规则（英文/中文）
- ✅ 字符估算（4 字符/1.5 汉字）

---

### 题 2：Token 如何计费？

**参考答案**：
> 主流 AI API 按 Token 数量计费，通常输出 Token 比输入 Token 更贵。例如 GPT-4 输入$0.03/1K Token，输出$0.06/1K Token。总费用 = 输入 Token×单价 + 输出 Token×单价。

**评分标准**：
- ✅ 计费模式（按 Token 计费）
- ✅ 价格差异（输出更贵）
- ✅ 计算公式

---

### 题 3：上下文窗口是什么？

**参考答案**：
> 上下文窗口是模型能处理的最大 Token 数。例如 GPT-4 支持 128K Token（约 100 万汉字），超过限制会丢失最早的内容。长文档需要分段处理或采用 RAG 技术。

**评分标准**：
- ✅ 定义（最大 Token 数）
- ✅ 超限影响（丢失内容）
- ✅ 解决方案（分段/RAG）
