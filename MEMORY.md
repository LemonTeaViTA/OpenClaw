| 反射缺点 | 3/5 | ✅ 掌握 | 🟢 已修复 |

**核心考点记录：**

**1. 方法重载：**
- 判断标准：只看参数列表（类型、顺序、个数）
- 不算重载：只有返回值不同、只有修饰符不同、参数名不同
- 编译报错场景：重复定义、仅返回值/修饰符不同

**2. try-catch-finally：**
- finally 中没有 return → 返回 try 中暂存的值
- finally 中有 return → 覆盖 try 的返回值
- finally 中修改变量 → 不影响返回值（已暂存）

**3. 反射：**
- 优点：运行时动态获取类信息、代码复用、框架基础
- 缺点：性能开销（慢 10-100 倍）、安全问题、代码可读性差、编译时不检查
- 场景：Spring/MyBatis 框架、动态代理、注解处理、IDE 功能

**核心考点记录：**

**1. 编译时 vs 运行时异常：**
- 编译时异常：Exception（非 RuntimeException），编译器强制检查，try-catch 或 throws
- 运行时异常：RuntimeException，编译器不检查，需修改代码修复
- 典型编译时：IOException、SQLException、ClassNotFoundException
- 典型运行时：NullPointerException、IndexOutOfBoundsException、ClassCastException

**2. 反射操作 API：**
- 获取 Class：`Class.forName()`、`类名.class`、`对象.getClass()`
- 获取字段：`getField()`（public）、`getDeclaredField()`（所有）
- 获取方法：`getMethod()`（public）、`getDeclaredMethod()`（所有）
- 暴力访问：`setAccessible(true)` 绕过访问检查
- 调用方法：`invoke(对象，参数...)`

**3. AIO 理解：**
- BIO：阻塞 IO，1 线程 1 连接，站在柜台等餐
- NIO：非阻塞 IO，1 线程多连接，不停问"好了吗"（轮询）
- AIO：异步 IO，操作系统回调，点餐后逛街，好了打电话通知
