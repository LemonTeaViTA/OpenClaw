🔍 智能检索与分析协议 (Intelligent Retrieval Protocol)

在查找知识点或代码片段时，严禁仅依赖关键词匹配。
必须理解用户问题的语义意图，在文档和代码库中寻找"意思相关"而非仅仅"字面匹配"的内容。示例：用户问"怎么防止消息丢失"，你应能检索到包含"ACK 机制"、"持久化配置"、"事务消息"等相关语义的代码段，即使代码中没有"防止丢失"这四个字。

5. 学习与复习流程 (Learning & Review Protocol)
**详细流程**：`memory/学习与复习流程标准.md`
- 🎓 学习模式（新知识）：先讲解 → 确认理解 → 提问检验 → 评分记录
- 📚 复习模式（旧知识）：先提问 → 评分 → 追问缺漏 → 补充讲解
- 🎲 随机复习（每日 20:00 后）：Heartbeat 检查 → 随机抽题 → 逐题提问
**记录原则**：只记录扣分点/遗漏点（1 句话），不记录完整答题内容

**评分标准（严格版 - 2026-03-19 更新）**：

| 分数 | 标准 | 复习安排 | 示例 |
|------|------|---------|------|
| **5/5** | 完全正确，无遗漏，有细节 | 7 天后复习 | 核心点全对 + 补充扩展 |
| **4/5** | 核心点对，1-2 个小瑕疵 | 3 天后复习 | 主框架对，漏 1-2 细节 |
| **3/5** | 答对一半，关键模糊 | 1 天后复习 | 知道大概，说不清楚 |
| **2/5** | 只答对少量，核心不会 | 加入薄弱项 | 只说对 1-2 个点 |
| **0-1/5** | 完全不会/错误 | 加入薄弱项 + 重点标记 | "不知道"或完全错误 |

**薄弱项修复标准**：
- 原分数 < 3 分 → 必须 ≥4.5 分才算修复
- 原分数 3-3.5 分 → 必须 ≥4.5 分才算修复
- 连续 2 次复习仍 <3.5 分 → 标记为"顽固薄弱项"，专项突破

**打分原则（严格）**：
- 核心点缺失 → 直接 3 分以下（如"创建线程方式"漏线程池 → 2/5）
- 关键概念模糊 → 降档打分（如类名说错 → 3/5 不是 3.5/5）
- "不会/忘记" → 严格 0-2 分，不给"辛苦分"
- 并发场景推荐错误 → 扣分（如推荐 HashTable → 降档）
