## 题目 1：Java 中异常处理体系？

**考察频率：** 🔥🔥🔥 高频

### 题目
Java 中异常处理体系？

### 标准答案要点

1. **Throwable 是基类**：所有错误和异常的父类
2. **两大分支**：
   - Error：程序无法处理的严重错误（OutOfMemoryError、StackOverflowError）
   - Exception：程序可以处理的异常
3. **Exception 分类**：
   - 编译时异常（Checked）：必须 try-catch 或 throws 声明（IOException、SQLException）
   - 运行时异常（Runtime）：编译器不强制处理（NullPointerException、IndexOutOfBoundsException）

### 面试考察点
- 异常层次结构理解
- Error vs Exception 区别
- 编译时异常 vs 运行时异常区别

## 题目 2：异常的处理方式？

**来源：** PDF 题 42  
**难度：** ⭐⭐⭐  
**考察频率：** 🔥🔥🔥 高频

### 题目
异常的处理方式？catch 和 finally 的异常可以同时抛出吗？

### 标准答案要点

1. **两种处理方式**：
   - 不处理，直接抛出（throw + throws）
   - try-catch 捕获处理

2. **catch + finally 同时抛异常**：
   - **不能同时抛出**，finally 中的异常会覆盖 catch 中的异常
   - 最终抛出的是 finally 中的异常，catch 中的异常被丢弃

3. **如何保留 catch 中的异常**：
   - 用变量暂存 catch 中的异常
   - 在 finally 中捕获并处理

### 面试考察点
- try-catch-finally 语法掌握
- 异常覆盖规则理解
- 实际异常处理经验

## 题目 3：try-catch-finally 执行流程

**来源：** PDF 题 43-1  
**难度：** ⭐⭐⭐⭐  
**考察频率：** 🔥🔥🔥 高频

### 题目
```java
public static int test() {
    try {
        return 1;
    } catch (Exception e) {
        return 2;
    } finally {
        System.out.print("3");
    }
}
```
输出结果是什么？

### 标准答案
**输出：`31`**

### 解析
1. try 块执行 `return 1`，准备返回 1
2. 没有异常，catch 不执行
3. finally 必执行，打印 `3`
4. 最后返回 try 中的值 `1`

### 面试考察点
- finally 执行时机理解
- return 与 finally 的执行顺序

## 题目 5：finally 修改变量是否影响返回值

**来源：** PDF 题 43-3  
**难度：** ⭐⭐⭐⭐⭐  
**考察频率：** 🔥🔥🔥 高频

### 题目
```java
public static int test1() {
    int i = 0;
    try {
        i = 2;
        return i;
    } finally {
        i = 3;
    }
}
```
输出结果是什么？

### 标准答案
**输出：`2`（不是 3！）**

### 解析
- 在执行 finally 之前，JVM 会先将 `i` 的结果**暂存起来**
- finally 执行完毕后，返回之前暂存的结果
- 所以即使 `i` 被修改为 3，最终返回的还是暂存的 `2`

### 面试考察点
- JVM 返回值暂存机制理解
- 基本数据类型 vs 引用类型在 finally 中的行为差异

## 题目 7：return 先执行还是 finally 先执行？

**来源：** 京东实习一面原题  
**难度：** ⭐⭐⭐⭐  
**考察频率：** 🔥🔥🔥 高频

### 题目
return 先执行还是 finally 先执行？

### 标准答案要点

**答案：finally 先执行**

**详细解释：**
1. 当 try 或 catch 中有 return 时，JVM 会先**暂存返回值**
2. 然后执行 finally 块
3. finally 执行完毕后，返回之前暂存的值

**注意：**
- 如果 finally 中也有 return，会覆盖之前的返回值
- 如果 finally 中修改了基本数据类型变量，不影响返回值（已暂存）
- 如果 finally 中修改了引用类型对象的属性，会影响返回对象的状态

## 题目 10：error 与 exception 异同，抛出 error 程序是无法运行的吗？

**来源：** 拼多多一面原题（PDF 题 41 收录）  
**难度：** ⭐⭐⭐⭐  
**考察频率：** 🔥🔥 中频

### 题目
error 与 exception 异同？抛出 error 程序是无法运行的吗？

### 标准答案要点

**相同点：**
- 都是 Throwable 的子类
- 都表示程序运行中的异常情况
- 都可以被 try-catch 捕获（理论上）

**不同点：**
| 对比项 | Error | Exception |
|--------|-------|-----------|
| 性质 | 严重错误 | 可处理异常 |
| 原因 | JVM 层面问题 | 程序逻辑或外部因素 |
| 可恢复性 | 通常无法恢复 | 可以捕获并处理 |
| 编译器要求 | 不强制处理 | Checked 必须处理 |
| 举例 | OutOfMemoryError | IOException |

**抛出 error 程序还能运行吗？**
- **抛出 Error 后，程序不一定会立即终止**
- 如果 try-catch 捕获了 Error，程序可以继续运行
- 但通常不建议捕获 Error，因为 JVM 状态可能已不稳定
- 某些 Error（如 ThreadDeath）会导致线程终止

### 面试考察点
- Error vs Exception 深度理解
- JVM 错误处理机制
- 实际工程经验
