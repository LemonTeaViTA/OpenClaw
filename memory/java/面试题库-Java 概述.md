**来源：** PDF 题 1

### 题目
什么是 Java？/ 多久开始学 Java 的？/ 平常用什么编程语言？

### 标准答案要点

**定义：** Java 是一门面向对象的编程语言，由 Sun 公司詹姆斯·高斯林团队于 1995 年推出

**特点：**
- 吸收了 C++ 的优点，抛弃了容易出错的地方（如指针）
- 自动垃圾回收
- 平台无关（一次编译，处处运行）

**跨平台：** 只需要在对应平台上安装 JDK，就可以在 Windows、macOS、Linux 上运行

## 题目 3：JVM、JDK、JRE 有什么区别？

**来源：** PDF 题 3 + 华为面试  
**难度：** ⭐⭐⭐⭐  
**考察频率：** 🔥🔥🔥 高频

### 题目
JVM、JDK、JRE 有什么区别？

### 标准答案要点

**包含关系：** JDK ⊃ JRE ⊃ JVM

| 名称 | 全称 | 作用 | 包含内容 |
|------|------|------|---------|
| **JVM** | Java 虚拟机 | 实现跨平台 | 将字节码转换为机器码并执行 |
| **JRE** | Java 运行时环境 | 运行 Java 程序 | JVM + 必需的库 |
| **JDK** | Java 开发工具包 | 开发 Java 程序 | JRE + 编译器 (javac) + 文档工具 (javadoc) + 字节码工具 (javap) 等 |

**简单说：** JDK 包含 JRE，JRE 包含 JVM

### 面试考察点
- Java 环境理解
- 开发工具了解

## 题目 4：什么是跨平台？原理是什么？

**来源：** PDF 题 4  
**难度：** ⭐⭐⭐  
**考察频率：** 🔥🔥🔥 高频

### 题目
什么是跨平台？原理是什么？

### 标准答案要点

**什么是跨平台：** Java 程序一次编译后，可以在多个操作系统上运行

**原理：**
- 增加了中间件 JVM
- JVM 负责将 Java 字节码转换为特定平台的机器码
- 不同操作系统有不同的 JVM 实现
- Java 程序不直接与操作系统交互，而是通过 JVM

### 面试考察点
- 跨平台原理理解
- JVM 作用理解

## 题目 5：什么是字节码？采用字节码的好处是什么？

**来源：** PDF 题 5  
**难度：** ⭐⭐⭐⭐  
**考察频率：** 🔥🔥🔥 高频

### 题目
什么是字节码？采用字节码的好处是什么？

### 标准答案要点

**什么是字节码：** Java 程序经过编译后产生的 **.class 文件**

**Java 程序执行三步：**
1. 编译：将源代码文件 .java 编译成 JVM 可以识别的字节码文件 .class
2. 解释：JVM 执行字节码文件，将字节码翻译成操作系统能识别的机器码
3. 执行：操作系统执行二进制的机器码

**好处：**
- 实现跨平台（字节码与平台无关）
- 提高安全性（JVM 可以检查字节码）
- 支持动态加载

### 面试考察点
- 字节码概念
- Java 执行流程

## 题目 6：为什么有人说 Java 是"编译与解释并存"的语言？

**来源：** PDF 题 6  
**难度：** ⭐⭐⭐⭐  
**考察频率：** 🔥🔥🔥 高频

### 题目
为什么有人说 Java 是"编译与解释并存"的语言？

### 标准答案要点

**编译型 vs 解释型：**
- 编译型：编译器针对特定操作系统，将源代码一次性翻译成机器码（C、C++）
- 解释型：解释器对源代码逐行解释成机器码并执行（Python、JavaScript）

**为什么 Java 是"编译与解释并存"？**
- Java 程序需要先将源代码编译成字节码文件（编译）
- 再解释执行字节码（解释）

**类比：**
- 编译型：找个翻译，等翻译将小说全部翻译成汉语，一次性读完
- 解释型：找个翻译，翻译一段读一段，慢慢把书读完
- Java：先写成标准格式（编译），再逐段翻译执行（解释）

### 面试考察点
- 编译 vs 解释理解
- Java 执行机制
