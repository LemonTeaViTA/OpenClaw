## 1. 泛型 (题 50)

### 泛型的作用

**核心作用：** 提高代码的类型安全

**好处：**
- 在编译时检查类型一致性
- 避免不必要的类型转换
- 避免 ClassCastException

**没有泛型的问题：**
```java
// 没有泛型，存储 Object 类型
List list = new ArrayList();
list.add("hello");
String str = (String) list.get(0);  // 必须强制类型转换
```

**有泛型：**
```java
List<String> list = new ArrayList<>();
list.add("hello");
String str = list.get(0);  // 无需强制转换
```

---

### 泛型的三种使用方式

#### 1. 泛型类

```java
// T 可以随便写，常见 T、E、K、V 等
public class Generic<T> {
    private T key;
    
    public Generic(T key) {
        this.key = key;
    }
    
    public T getKey() {
        return key;
    }
}

// 实例化泛型类
Generic<Integer> genericInteger = new Generic<Integer>(123456);
```

#### 2. 泛型接口

```java
// 定义泛型接口
public interface Generator<T> {
    public T method();
}

// 实现泛型接口
class GeneratorImpl<T> implements Generator<String> {
    @Override
    public String method() {
        return "hello";
    }
}
```

#### 3. 泛型方法

```java
// 泛型方法（<E> 在返回值前）
public static <E> void printArray(E[] inputArray) {
    for (E element : inputArray) {
        System.out.printf("%s ", element);
    }
    System.out.println();
}

// 使用
Integer[] intArray = {1, 2, 3};
String[] stringArray = {"Hello", "World"};
printArray(intArray);
printArray(stringArray);
```

---

### 泛型常用通配符

| 通配符 | 含义 | 举例 |
|--------|------|------|
| **?** | 不确定的 Java 类型 | List<?> |
| **T (type)** | 具体的一个 Java 类型 | Generic<T> |
| **K V (key value)** | 键值对的 Key 和 Value | Map<K,V> |
| **E (element)** | 元素 | List<E> |

---

### 泛型擦除（类型擦除）

**定义：** Java 的泛型是**伪泛型**，编译期间所有类型信息都会被擦掉

**本质：**
- 泛型只存在于源码中
- 编译时进行类型检查
- 运行时没有泛型信息

**示例：**
```java
LinkedList<Cat> cats = new LinkedList<Cat>();
LinkedList list = cats;  // 去掉泛型，但是同一个链表
list.add(new Dog());  // 完全没问题！运行时不检查
```

**为什么需要类型擦除？**
- **向下兼容**：JDK5 之前没有泛型
- 为了让 JVM 保持向下兼容

## 2. 注解 (题 51)

### 注解的本质

**定义：** Java 注解本质上是一个**标记**

**类比：** 生活中一个人的小装扮（戴帽子、戴眼镜）

**特点：**
- 可以标记在类上、方法上、属性上
- 标记自身可以设置值（如帽子颜色是绿色）

---

### 注解的作用

**有了标记后：** 在编译或运行阶段识别这些标记，然后搞一些事情

**应用场景：**
1. **AOP**：使用注解作为切点（运行期注解应用）
2. **Lombok**：注解在编译期运行（自动生成代码）
3. **Spring**：@Autowired 等注解

---

### 注解的生命周期（三大类）

| 生命周期 | 说明 | 举例 |
|---------|------|------|
| **SOURCE** | 给编译器用的，不会写入 class 文件 | @Override |
| **CLASS** | 会写入 class 文件，类加载阶段丢弃（运行时没有） | - |
| **RUNTIME** | 会写入 class 文件，永久保存，可通过反射获取 | @Autowired |

**举例说明：**

**@Override（SOURCE）：**
```java
@Override
public String toString() {
    return "hello";
}
```
- 给编译器用的，检查是否真的重写了父类方法
- class 文件中不会有 Override 标记

**@Autowired（RUNTIME）：**
```java
@Autowired
private UserService userService;
```
- RUNTIME 生命周期
- 运行时可以通过反射获取注解信息
- 可以拿到标记的值（如 required）

## 3. 反射 (题 52)

### 什么是反射？

**定义：** 反射允许 Java 在**运行时**检查和操作类的方法和字段

**作用：** 动态地获取类的字段、方法、构造方法等信息，并在运行时调用方法或访问字段

---

### 反射 vs 普通创建对象

**普通方式（编译时确定）：**
```java
Person person = new Person();
// Person 类的信息在编译时就确定了
```

**反射方式（运行时确定）：**
```java
// 编译期无法确定类的信息，运行时获取
String className = "java.util.Date";
Class<?> cls = Class.forName(className);
Object obj = cls.newInstance();
```

---

### 反射的实现类

**主要类：**
- `java.lang.Class`：核心类
- `java.lang.reflect` 包：
  - `Method`：方法
  - `Field`：字段
  - `Constructor`：构造方法

---

### 反射的基本操作

#### 1. 动态加载类并创建对象

```java
String className = "java.util.Date";
Class<?> cls = Class.forName(className);
Object obj = cls.newInstance();
System.out.println(obj.getClass().getName());
```

#### 2. 访问方法

```java
// 加载并实例化类
Class<?> cls = Class.forName("java.util.Date");
Object obj = cls.newInstance();

// 获取并调用方法
Method method = cls.getMethod("getTime");
Object result = method.invoke(obj);
System.out.println("Time: " + result);
```

#### 3. 访问字段

```java
// 访问字段
Field field = cls.getDeclaredField("fastTime");
field.setAccessible(true);  // 私有字段需要这样做
System.out.println("fastTime: " + field.getLong(obj));
```

---

### 反射的应用场景

#### 1. Spring 框架

**用途：** 动态加载和管理 Bean

```java
Class<?> clazz = Class.forName("com.example.MyClass");
Object instance = clazz.newInstance();
```

#### 2. 动态代理（AOP）

**用途：** 创建代理类，运行时动态处理方法调用

```java
InvocationHandler handler = new MyInvocationHandler();
MyInterface proxyInstance = (MyInterface) Proxy.newProxyInstance(
    MyInterface.class.getClassLoader(),
    new Class<?>[] { MyInterface.class },
    handler
);
```

#### 3. 测试框架（JUnit、TestNG）

**用途：** 发现和执行测试方法

```java
// 扫描类，查找带有@Test 注解的方法
Method testMethod = testClass.getMethod("testSomething");
testMethod.invoke(testInstance);
```

---

### 反射的原理

**Java 程序执行流程：**
1. **编译**：生成字节码 (.class) 文件
2. **运行**：JVM 类加载，将类型信息加载进方法区
3. **反射**：获取这些信息，进行各种操作

**本质：** 反射就是去获取 JVM 方法区中的类型信息
