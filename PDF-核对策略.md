## 🎯 核心原则

1. **分部分识别**：每次处理 5-8 题，避免一次性处理大量内容导致信息丢失
2. **PDF 优先**：PDF 上的知识点题目是核心，确保一个不漏
3. **三次核对**：学习前、学习后、复习时都要核对
4. **及时记录**：学习/复习过程中立即记录进度和状态
5. **补充原则**：PDF 知识点为基础，重要知识点可补充额外题目

---

## 📋 三次核对流程

### 第 1 次核对（学习前）

**目的：** 提取知识点清单，确保无遗漏

**方法：**
```bash
# 1. 提取 PDF 文本
pdftotext "XXX.pdf" /tmp/xxx.txt

# 2. 分部分提取（按题号）
# 第 1 部分：题 1-8
sed -n '/题 1/,/题 8/p' /tmp/xxx.txt > /tmp/xxx-part1.txt

# 第 2 部分：题 9-16
sed -n '/题 9/,/题 16/p' /tmp/xxx.txt > /tmp/xxx-part2.txt

# 3. 列出题目清单
grep "题 [0-9]*\." /tmp/xxx-part1.txt

# 4. 对照知识点清单.md 检查是否有遗漏
```

**输出：** 完整题目列表 + 知识点清单.md 更新

---

### 第 2 次核对（学习后）

**目的：** 确认当次学习的题目是否全部讲解

**方法：**
```bash
# 1. 按同样分部分方式提取 PDF
# 2. 逐一确认每个题目是否已讲解
# 3. 发现遗漏立即补充
# 4. 更新知识点清单.md 的 [x] 标记
```

**检查清单：**
- [ ] 所有题目已讲解
- [ ] 知识点清单.md 已更新
- [ ] MEMORY.md 已记录答题情况
- [ ] Obsidian 笔记已更新（纯净知识点）

---

### 第 3 次核对（复习时）

**目的：** 利用大模型判断知识点掌握程度，发现隐蔽遗漏

**方法：**
```
1. 按同样分部分方式提取 PDF
2. 将一部分题目 + 用户答案 + 笔记发给大模型
3. 让大模型判断：答案是否完整、是否有理解偏差
4. 分多次完成整个章节的核对
```

**时机：**
- 第 1 次复习（2 天后）：分部分核对
- 第 2 次复习（7 天后）：分部分核对

## 📊 分部分大小标准

| 章节大小 | 分部分方式 | 每部分题数 | 核对次数 |
|---------|-----------|-----------|---------|
| 小章节（≤8 题） | 不分部分 | 全部一次处理 | 1 次 |
| 中章节（9-16 题） | 分 2 部分 | 5-8 题/部分 | 2 次 |
| 大章节（17-30 题） | 分 3-4 部分 | 5-8 题/部分 | 3-4 次 |
| 超大章节（>30 题） | 分 5+ 部分 | 5-8 题/部分 | 5+ 次 |
